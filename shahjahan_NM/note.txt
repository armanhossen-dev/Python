What I integrated:
x-axis linear acceleration from Accelerometer.csv (696 samples, ~10ms spacing, ~6.98s span) 
— integrating acceleration over time gives velocity change.

Setup: 
Simpson's 1/3 needs an even number of intervals, 
Simpson's 3/8 needs a multiple of 3. 
I resampled the (near-uniform) data onto a uniform grid with 690 intervals 
(h ≈ 0.01004 s) — divisible by both — so all three rules run on the identical dataset.

Integration results (Δv, m/s):

Method	Result
Trapezoidal	-0.007872
Simpson's 1/3	-0.008006
Simpson's 3/8 (reference)	-0.008826

Error analysis (Simpson 3/8 as truth):

Method	Absolute Error	Relative Error	% Error
Trapezoidal	0.000955	0.1082	10.82%
Simpson's 1/3	0.000820	0.0930	9.30%

Simpson's 1/3 is closer to the 3/8 result than Trapezoidal, as expected since both Simpson variants are higher-order (O(h⁴)) than Trapezoidal (O(h²)).

The script is in the file above — it's parameterized so you can point it at any of the other sensor CSVs (Gyroscope, TotalAcceleration, etc.) or a different axis just by changing the file path and column name.
